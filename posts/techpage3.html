智慧树刷课脚本推荐及其嚼碎食用指南

  • 最近从舍友那里认识到了一个存在于github上面的救命脚本---智慧树刷课脚本，直接让我解放设备！二话不说，直接帮你一步到位上手！
  • 找寻：github网站搜索: fuckZHS ,然后选择第一个搜索结果
  • 寻找作者的教程指南：进入他的仓库，点击查看README.md,里面有他的操作指引指北

好！现在我就帮你把指北里的准备工作给你讲明白

  • 1.首先，打开终端bash，检查你的python版本（一定要3.10以上）。如果你不知道电脑安装了什么版本, Bash输入：法1：python --version 或法2（win特有）：py -0 没有就装
  • tips:3.14-(alpha)版本的话不行，因为到今天它都没发布，用了有大问题（别问为什么，呜呜~）
  • 2.接下来先跟着我操作：回到作者的仓库界面（如果你听不懂的话，那就是你在搜索完fuckZHS后，点击人家的名字所进入到的界面）， 会发现有个大绿按键Code，哎！你点击Code旁边的展开，看到最底下的Download Zip，点击把仓库里的所有文件压缩包下载下来，然后解压。这里推荐放到桌面便于操作。
  • 3.然后bash运行准备工作的“pip install -r requirements.txt”
  • 4.为了脚本能成功进入zhihuishu,我们要给用户名及密码给脚本。bash运行： notepad config.json .记事本或VScode打开它,找到username和password字段 输入保存在fuckZHS文件夹
  • 5.再复制fuckZHS文件夹的地址，在bash上运行：cd 地址 （引号去掉），这是进入到文件里。
  • 6.最后bash执行： python main.py (如果你的python版本原本没什么问题的话)
  • 7.成功标志：bash最终运行生成二维码，用手机设备扫进授权即可开始刷课fucking

关于我能遇见的问题解惑（看一眼）

  • 1.如果python版本是3.14-alpha的话有什么问题？答： alpha 版本不稳定：可能存在兼容性问题、API 变动、标准库行为异常 第三方库（如 requests）可能未适配 3.14 脚本静默退出，很可能是某个库在 3.14 alpha 下初始化失败但未报错。。。我就是因为用了这个版本，导致运行上述5.6步时bash一直没反应，处于静默状态，浪费我许多时间去找问题 怎么解决：如果你电脑有其他版本符合条件，你可以在bash上指定py版本运行，改为： py -版本 main.py 没有其他版本的话去下
  • 2.对某些自然现象，不懂的好好把人家的README文章看看
  • 3.如果你第一次刷课成功了，那下一次只需在bash上重复执行5.6步即可，有时它不弹二维码直接开始刷了
  • 4.刷课默认为1.5倍速，从头刷，考试还是要做的。
  • 5.总之，以作者的准备工作为中心展开就行。遇到其他问题问一下AI
